# Claude Design 프롬프트 모음 — WSWG 전체 화면 (S1~S9 + 관리자)

> **용도**: Claude Design은 우리 레포를 못 보므로 프롬프트는 **자체 완결형**이어야 한다.
> **사용법**: 한 화면을 디자인할 때 **[§0 공통 제약]** + **[해당 화면 §N]** 을 이어붙여 하나의 프롬프트로 붙여넣는다.
> **근거(실물)**: `wswg_frontend` 실제 토큰(`src/assets/styles/globals.css`)·컴포넌트(`src/components/ui/*`, `src/features/trip/*`, `src/layouts/AppShell`) · `화면정의서.md` · `디자인시스템.md` v0.3.
> **원칙**: 아래 토큰·컴포넌트 어휘를 **벗어나지 말 것(강제)**. 임의 색/그림자/폰트 추가 금지.

## §0 공통 제약 (모든 프롬프트 앞에 붙이기)

```
한국어 협업형 여행 플래너 "어디갈래?(WSWG)"의 화면을 디자인해줘.
데스크탑 웹앱(노션 스타일) + 모바일 반응형. 아래 디자인 시스템을 그대로 강제하고, 벗어나지 마.

[기술/렌더 가정]
Vue 3 + Tailwind CSS 4 + reka-ui(shadcn-vue 패턴) + Pretendard. 이 시스템에 그대로 옮길 수 있는 구조로 출력.
색·간격은 아래 토큰만 사용. 임의 색·그림자·그라데이션 금지(평면 우선, 그림자는 모달/팝오버에만).

[디자인 토큰 — 정확히 이 값만]
폰트: Pretendard. 무게로 위계(Page Title 800, 헤딩 700, 본문 400~500).
표면: 배경 #FFFFFF · 사이드바 #FBFBFA · sunken/muted #F7F6F3 · hover/accent #F1F1EF · 카드 #FFFFFF.
보더(헤어라인): #EDEDEB(기본), #E3E2DF(입력/카드 강).
글자: 본문 #37352F · 보조 #6B6A65 · 흐림 #9B9A95.
브랜드(버튼·선택·링크·활성탭에만, 면적 최소): #2C6FE3 / 흰 글자, 포커스 ring #2C6FE3.
위험: #C9453B.
블록 타입 칩(배경/글자): 관광 #E3EEF8/#1F5C8F · 식당 #FBEADB/#9A5B17 · 이동 #EDEDEB/#605F5A · 숙소 #EEE7F6/#6A4B97 · 메모 #FAF1D2/#86691A · 기록 #E5F0E4/#3F6B43.
협업 카렛 색(라운드로빈): #2C6FE3 · #E0633D · #3F9A6A.
radius: 8px(카드·버튼) / 5px(칩·pill·입력) / 12px(모달). 간격은 4px 그리드(4·8·12·16·24·32).

[고정 컴포넌트 어휘 — 이 생김새 그대로]
- AppShell: 좌측 사이드바 228px + 상단 토픽바 + 본문 캔버스(전체 높이).
  · 사이드바(#FBFBFA): 워크스페이스 스위처 → 홈/검색/지도 메뉴(NavItem) → 페이지 트리(모임 › 여행 › Day, 이모지 아이콘). 활성 항목 accent 배경.
  · 토픽바: 브레드크럼 + (해당 시)LiveIndicator + AvatarStack + "공유" 버튼.
- Button: Primary=#2C6FE3/흰글자, 보조=accent 배경/ink 글자, 위험=#C9453B. radius 8.
- BlockTag: soft 색칩 "이모지 + 라벨"(📍관광·🍜식당·🚄이동·🏨숙소·📝메모·✓기록), radius 5, 작은 글자, 색만으로 구분 금지(이모지+라벨 항상).
- PropertyPill: 인라인 작은 칩(accent 배경·흐린 글자), 예 "💰 9,000원" "⭐ 5.0".
- PlaceCard: 좌측 46px 정사각 썸네일(없으면 옅은 블루-베이지 그라데이션) + 제목 + (테마)BlockTag + 보조 meta(주소·⭐·휴무) + 우측 액션 슬롯("＋ 담기"). radius 8, 헤어라인 보더, hover 시 accent.
- TripBlock(일정 레일형): 좌측 수직 타임라인 레일에 타입색 점 + hover 드래그핸들(⋮⋮) + 선두 이모지 + 오버라인 시간("10:00 · 1시간", 대문자·자간 넓게·흐림) + 제목(+협업 카렛) + 그 아래 BlockTag·지역 meta·PropertyPill·"직접 추가" + 하단 미디어 썸네일(48×64, 동영상 ▶ 오버레이).
- CollabCaret: 컬러 세로 막대 + 이름표 "│민지"(협업 색).
- LiveIndicator: 초록(#3F9A6A) 점 + 펄스 애니메이션 + 라벨("실시간 동기화" / "N명 편집 중").
- AvatarStack: 겹친 원형 이니셜 아바타(협업 색 테두리).
- Tabs: 활성 탭 밑줄 #2C6FE3.
- FilterChips: 토글 칩 묶음(선택 = #EAF1FD 배경 + #2C6FE3 글자/보더).
- SearchInput / Input: 헤어라인 보더, sunken 배경 가능, 좌측 돋보기.
- Dialog: 데스크탑 = 중앙 모달(흰 배경·radius 12·옅은 그림자) / 모바일 = 풀스크린 또는 바텀 시트.
- DropdownMenu: 카드·행의 ⋮ 더보기.
- Sonner(Toast): 우하단 알림.
- Badge: 상태/카운트 작은 라벨.

[반응형 — 강제]
데스크탑 ≥1024: 풀 셸. (S6 여행 편집만 데스크탑 풀편집 우선, 모바일은 조회+간단편집.)
태블릿 768~1024: 사이드바 아이콘 축소.
모바일 <768: 사이드바 → 햄버거 off-canvas drawer · 패널/필터 → 바텀시트 · 모달 → 풀스크린 시트 · 카드 1열 · 가로 캐러셀은 스와이프. 터치 타깃 ≥44px, 호버 전용 액션은 항상 보이는 +/⋮ 버튼으로.

[카피·보이스 — 강제]
해요체. 친근·명료·들뜬·든든. 반말·사무적·허세·압박 금지.
예) 빈 상태 "아직 ~없어요! ~해볼까요?" · 에러 "잠깐, 길을 잃었어요. 다시 해볼까요?" · 협업 "민지님이 점심 일정을 바꿨어요".

[출력]
데스크탑 메인 화면 + 모바일 변형을 함께. 한국어 UI 텍스트. 위 토큰·컴포넌트 어휘 밖으로 나가지 마.
```

## §1 — S1 랜딩 `/`

```
화면: 랜딩(게스트 진입). 셸은 간소화(좌측 사이드바 없이, 상단 글로벌 헤더 + 로그인 버튼).
구성(데스크탑, 세로 스택):
1) 글로벌 헤더: 로고 + 우측 "둘러보기" 보조 버튼 · "시작하기" Primary.
2) Hero: 큰 헤드라인 "고르기만 하세요, 일정은 저희가 짜드려요" + 서브 "지역이랑 취향만 정하면, 친구들이랑 같이 일정을 다듬을 수 있어요." + "시작하기"(Primary)/"관광지 둘러보기"(보조).
3) 인기 여행지: 섹션 타이틀 "요즘 다들 여기로 떠나요" + PlaceCard 가로 캐러셀(썸네일·이름·지역·"N개 여행에 담겼어요" 카운트).
4) 서비스 소개: 3개 가치 카드(자동 일정 / 실시간 공동 편집 / 관광지 검색), 평면.
5) 하단 CTA + 푸터.
모바일: 세로 스택, 인기 여행지는 스와이프 캐러셀(카드 1.2장 보이게).
상태: 인기 여행지 로딩=카드 Skeleton, 비면 폴백(이미지 있는 관광지)으로 채움.
```

## §2 — S2 로그인 `/login`

```
화면: 소셜 로그인(게스트). 간소화 셸(상단 로고만). ⚠️ 이메일/비밀번호 폼 없음(폐기).
구성(데스크탑): 본문 중앙 로그인 카드(radius 12) — 헤드라인 "어디갈래에 오신 걸 환영해요! 소셜 계정으로 바로 시작해요" + "구글로 시작하기" 버튼 + "카카오로 시작하기" 버튼(각 브랜드 식별색+아이콘+라벨, 두 버튼 동등 강조) + 약관 마이크로카피 "로그인하면 이용약관과 개인정보 처리방침에 동의하는 것으로 볼게요".
모바일: 1열, 두 버튼 풀폭 스택(터치 44px).
상태: 클릭 후 로딩 "잠깐만요, 로그인하고 있어요…" / 실패 Toast "로그인이 잘 안 됐어요. 다시 해볼까요?".
```

## §3 — S3 관광지 검색·목록 `/attractions`

```
화면: 관광지 검색(게스트). 간소화 셸.
구성(데스크탑): 
- 검색바: SearchInput(키워드) + 시/도 Select + 구/군 Select(시도 선택 시 활성) + "검색" Primary.
- 필터 칩: 콘텐츠타입 테마 FilterChips(다중 토글) + 우측 결과 수.
- 결과: PlaceCard 그리드 3~4열(썸네일·제목·지역·테마 BlockTag).
- 하단 페이징.
모바일: 키워드 + "필터" 버튼 → 시/도·구/군·테마칩을 바텀시트로. 카드 1열.
상태: 빈 결과 EmptyState "찾는 곳이 없네요. 키워드나 지역을 바꿔볼까요?" + "필터 초기화" / 로딩 카드 Skeleton.
```

## §4 — S4 관광지 상세 `/attractions/{id}`

```
화면: 관광지 상세(게스트 열람, "내 여행에 추가"는 회원). 간소화 셸 + 브레드크럼.
구성(데스크탑):
- 히어로: 대표 이미지(16:9) + 제목(Page Title)·카테고리 BlockTag(뉴트럴 1색)·주소.
- 2열(좌:우 ≈ 2:1): 좌 = overview 본문 · 전화/홈페이지 연락 리스트 / 우(sticky) = "내 여행에 추가" Primary → 지도(단일 핀) → 주소 요약.
모바일: 1열 스택(이미지→제목→연락→overview→지도). "내 여행에 추가"는 하단 고정 액션바 → 누르면 대상 여행 선택 바텀시트.
상태: 빈 필드 행 숨김 / 이미지 없으면 옅은 플레이스홀더 "아직 사진이 없어요." / 404 EmptyState "찾는 관광지가 없어요. 다른 곳을 둘러볼까요?" + "검색으로 가기" / 게스트가 "추가" 클릭 → "내 여행에 담으려면 로그인이 필요해요".
```

## §5 — S5 여행 자동 생성 `/plans/new`

```
화면: 여행 자동 생성(회원). 앱 셸 + 토픽바 브레드크럼 "모임명 › 새 여행 › 자동 생성".
구성(데스크탑): 중앙 폼 카드, 2열 그리드 —
- 1열: 지역(시/도 Select → 구/군 Select), 기간(출발일·종료일 date).
- 2열: 인원(stepper), 여행 스타일(다중 선택 칩 그리드 — 선택 시 #EAF1FD 배경+#2C6FE3).
- 하단 액션바: "자동 생성 ✨" Primary + "취소" + 폼 요약 한 줄("부산 · 2박3일 · 2명 · 자연·도보").
모바일: 1열 폼, 스타일 칩 2열 wrap, "자동 생성 ✨"는 하단 고정 풀폭.
상태: 생성 중 폼 위 진행 인디케이터 오버레이 "일정을 짜고 있어요…" / 종료일<출발일 → 인라인 검증(보더 #C9453B) "돌아오는 날이 떠나는 날보다 빨라요. 다시 골라볼까요?" + 버튼 비활성 / 후보 부족 partial 안내.
헤드라인 "어디로, 어떤 여행을 떠나볼까요?".
```

## §6 — S6 여행 편집(협업) `/trips/{id}` ⭐핵심

```
화면: 여행 편집(협업, 멤버). 앱 셸. **데스크탑 풀편집 우선.** 노션식 블록 에디터, 📅 일정 뷰 기본.
구성(데스크탑):
- 토픽바: 브레드크럼 "부산크루 / 부산 2박3일" + LiveIndicator(🟢 "실시간 동기화") + AvatarStack(동행인 3명) + "공유".
- 페이지 헤더: 이모지 + 큰 타이틀 "부산 2박3일" + 속성 테이블 한 줄(📅 기간 · 📍 지역 · 👥 동행 AvatarStack · 🎨 스타일 · 💰 예산).
- 뷰 탭(Tabs): 📅 일정(활성, 밑줄 #2C6FE3) / 🖼️ 갤러리 / 🗺️ 지도 / 📋 보드.
- 일정 뷰 본문: "Day 1 · 7/1 화" 섹션들. 각 Day에 수직 타임라인 레일 + 구간 노드(🌅 오전 / ☀️ 오후 / 🌆 저녁). 레일에 TripBlock 카드들이 매달림(타입색 점·이모지·오버라인 시간 "10:00 · 1시간"·제목·BlockTag·PropertyPill 💰⭐🎫·미디어 썸네일). "이동" 타입은 카드 대신 앞 블록 뒤 회색 스트립(🚄). 블록 hover 시 좌측 드래그핸들(⋮⋮)+"+", "/" 입력 시 슬래시 메뉴(타입 선택). 한 블록은 편집중: 옅은 색 배경 + CollabCaret "│민지".
  예시 블록: 해운대 해수욕장(관광·10:00·1시간·⭐5.0·사진 2장) / 속씨원한 돼지국밥(식당·12:30·💰9,000원) / KTX(이동·15:00) / "야경 동선 다시 보기"(메모·시간 미정).
모바일(<768): 조회+간단편집. 사이드바→drawer, 속성 테이블→접힘 시트, 뷰 탭 가로 스크롤(📋 보드 제외, "보드는 큰 화면에서 편해요" 칩), 블록 풀폭 카드 + 항상 보이는 +/⋮, 사진 추가는 카메라/갤러리.
상태: 빈 "아직 일정이 비어 있어요! 첫 블록을 더해볼까요?" / 로딩 Skeleton / 끊김 시 LiveIndicator 회색 "연결이 끊겨 다시 잇는 중…".
협업 알림 Toast "민지님이 점심 일정을 바꿨어요".
```

## §7 — S7 마이페이지 `/mypage`

```
화면: 마이페이지(회원). 앱 셸.
구성(데스크탑): 페이지 헤더 "내 여행" + "+ 새 여행 만들기" Primary. 그 아래 Tabs(내 여행(N) / 참여중 여행(N)). 본문 = 여행 카드 그리드 3열 — 제목·기간·인원(AvatarStack)·상태 Badge·썸네일·우상단 ⋮ DropdownMenu(삭제).
상태 Badge 색: 예정=관광 blue(#E3EEF8/#1F5C8F) · 진행중=메모 yellow(#FAF1D2/#86691A) · 완료=기록 green(#E5F0E4/#3F6B43). 점 아이콘+라벨 병행.
모바일: 카드 1열, 탭 상단 고정, "+ 새 여행"은 풀폭 또는 우하단 FAB.
상태: 빈(내 여행) "아직 계획이 없어요! 첫 여행을 만들어볼까요?" + CTA / 빈(참여중) "아직 함께하는 여행이 없어요. 친구의 모임에 들어가보세요" / 삭제 확인 Dialog "이 여행을 삭제할까요? 일정은 사라지지만, 함께 남긴 추억 사진은 모임 지도에 그대로 있어요".
```

## §8 — S8 발자취 지도 `/groups/{id}/map` ⭐핵심

```
화면: 그룹 발자취 지도(멤버). 앱 셸 + 토픽바 "모임 ▸ 발자취 지도".
구성(데스크탑): 좌 = 전국 지도(시도/구군 GeoJSON 권역 폴리곤 — 다녀온 권역은 #EAF1FD 채움+#2C6FE3 보더, 미방문은 #F7F6F3+헤어라인 / 지역마다 대표 추억 핀). 좌상단 줌 토글 + 요약 "전국 17곳 중 5곳, 같이 다녀왔어요". 우 = 대표 추억 패널(320px): 지역별 카드(썸네일·지역명·미디어 타입 배지 📷🎙️🎬·"대표 변경").

상태: 빈 "아직 함께 다녀온 곳이 없어요. 여행을 다녀오면 여기에 발자취가 쌓여요." / 로딩 "발자취를 불러오고 있어요…".
헤드라인 "우리가 함께 밟은 발자취예요". (다녀온 여행만 노출, 색칠은 프론트 GeoJSON.)
```

## §9 — S9 모임 관리 `/groups`
