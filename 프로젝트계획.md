## 0. 전략 요약 (이 일정이 성공하는 4가지 조건)

1. **트랙 분담** — 풀스택 2명이므로 레이어가 아닌 **기능 수직 분담**(각자 end-to-end). 통합 마찰 최소화.
2. **PostgreSQL 마이그레이션 선행** — 모든 기능의 전제. **D1–D2에 끝낸다.** 늦어지면 전체가 막힘.
3. **기능별 MVP 깊이 고정** — "전부 하되, 각 기능은 데모 가능한 최소 깊이로"(§4). 깊이 욕심이 일정을 깨는 1순위 리스크.
4. **6/19 Feature Freeze** — 이후(6/20~23)는 신규 기능 금지, **통합·QA·배포·발표 준비만**.

## 4. 기능별 MVP 깊이 (욕심 차단선 — 이 선까지만)

| 기능 | 이번 범위(O) | 후속/제외(X) |
|------|--------------|--------------|
| 로그인 | 소셜(구글/카카오)+JWT (기구현) | 이메일/비번 |
| 관광지 | 지역·키워드·타입 필터, 상세, 페이징 | 고급 추천·랭킹 |
| 자동생성 | **규칙 기반**(지역+스타일→일자/시간 배치) | 동선·거리 최적화, ML |
| 실시간 협업 | **항목 단위 last-write-wins** + 버전 충돌 감지 + 프레즌스 | CRDT/OT |
| 멀티미디어 지도 | **사진 필수** + 권역 색칠 + 마커 팝업 | 음성·영상은 시간 남으면(D11) |
| 마이페이지 | 내/참여 여행 목록 + 상태 태그 | 통계·대시보드 |

> **컷 우선순위(시간 부족 시 순서대로 축소)**: ① 영상 → ② 음성 → ③ 프레즌스 표시 → ④ 자동생성 시간 슬롯 정교화. **사진 발자취 지도·실시간 편집 코어는 사수.**

## 6. 협업 규칙

- **일 1회 스탠드업**(10분): 어제/오늘/블로커.
- **브랜치**: `feat/<기능>` → PR → 1일 1회 main 머지. main 항상 빌드 가능.
- **API 컨트랙트 우선**: 스키마 합의 후 각자 mock으로 병렬 개발.
- **머지 게이트**: 기능 머지 전 해당 플로우 수동 동작 확인.
- **환경변수**: OAuth secret·JWT secret·DB·스토리지 키는 `.env`/환경변수 (하드코딩 금지).
