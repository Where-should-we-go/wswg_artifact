**프로젝트명**: 어디갈래? (Where Should We Go, **WSWG**)
**한 줄 정의**: 지역·스타일을 선택하면 데이터 기반으로 일정을 **자동 생성**하고, 모임원과 **실시간 공동 편집**하며, 다녀온 여행을 **멀티미디어 지도**로 기록하는 협업형 여행 플래너

**작성일**: 2026-06-09 / **개정**: 2026-06-23 — `trip_attractions`·`trip_region_snapshots`·`trip_media` 제거, `trips.data` JSONB + `group_region_media`로 통합(ERD.md §5)
**대상 시스템**: `com.ssafy.wswg` (Spring Boot)
**근거 문서**: `artifact/README.md`, `artifact/usecase.md`, `artifact/wireframe.html`, `docs/ERD.md`(데이터 모델 정본)

## 1. 개요

### 1.1 서비스 정의
여행 준비의 번거로움을 줄이는 협업형 여행 플래너. 사용자가 **지역 + 여행 스타일**을 입력하면 한국관광공사 TourAPI 데이터를 기반으로 **날짜·시간별 일정을 자동 생성**하고, 모임원이 **Google Docs 스타일로 실시간 공동 편집**한다. 다녀온 여행은 **사진·음성·영상**과 함께 **지도에 발자취로 누적**된다.

### 1.2 핵심 가치
- **시간 단축**: 데이터 기반 자동 일정 생성으로 계획 시간 최소화
- **협업 경험**: 모임 단위 실시간 공동 편집으로 의사결정 단순화
- **추억 기록**: 다녀온 여행을 멀티미디어 지도로 되돌아봄 (차별화 기능)

### 1.3 핵심 기능 (artifact 기준 + 격상 항목)
1. **여행 계획 자동 생성** — 지역/기간/인원/스타일 → 일정 추천 ⭐
2. **실시간 공동 편집** — 모임원과 일정 동기 편집 ⭐
3. **관광지 정보 제공** — 시도/구군 + 키워드 검색, 상세
4. **마이페이지** — 내 여행 / 참여중 여행 관리
5. **멀티미디어 발자취 지도** — 그룹이 다녀온 여행을 사진·음성·영상과 지도에 기록 ⭐ (핵심 격상)

### 1.4 액터 (usecase.md)
| 액터 | 설명 |
|------|------|
| 게스트(Guest) | 비로그인 방문자. 관광 정보 열람·가입 가능 |
| 회원(User) | 로그인 사용자. 여행 생성/수정/공유 |
| 동행인(Companion) | 모임에 초대받아 공동 편집에 참여 |
| 관리자(Admin) | 관광지 마스터 데이터 관리 |
| 한국관광공사 API | 관광지 원천 데이터 외부 시스템 |

## 2. 기술 스택 / 환경

| 구분 | 내용 | 비고 |
|------|------|------|
| Backend | Spring Boot, MyBatis | 코드 현황 기준(MyBatis) |
| 인증 | **OAuth2 소셜 로그인(Google/Kakao)** + JWT(access 20분/refresh 14일) | 기존 구현 |

| 외부 연동 | 한국관광공사 TourAPI, 지도 SDK, 오브젝트 스토리지(S3/MinIO) | |
| Frontend | `wswg_frontend` (Vue/React) | 별도 |

## 3. 제약사항 (Constraints)

| ID | 제약사항 | 영향 |
|----|----------|------|
| CON-01 | **MySQL → PostgreSQL 마이그레이션.** 멀티미디어 JSONB·PostGIS·집계 요구로 PostgreSQL 단일 통일. | DataSource/방언/매퍼 |
| CON-02 | 로그인은 **소셜 OAuth2** 기준. 와이어프레임의 이메일/비밀번호 화면은 폐기. | 회원가입 흐름 |
| CON-03 | 협업은 **모임/그룹 모델**(`groups`/`user_group`) 기준. 공유 링크는 모임 초대 수단으로만 보조 사용. | 데이터 모델 |
| CON-04 | `users` 스키마는 현행 `UserDto`(`id/name/email/profileImageUrl/role/createdAt`) 기준. | ERD 통일 |
| CON-05 | "여행 스타일"은 `contenttypes`(콘텐츠 타입) 선호로 매핑하여 자동 생성 입력으로 사용. 별도 theme 엔티티 없음. | 자동 생성 알고리즘 |
| CON-06 | `trips`의 `user_id`/`group_id`는 XOR(정확히 하나). | 무결성 |
| CON-07 | "다녀온(완료) 여행"은 `end_date < 오늘` 자동 판정. | 발자취 조회 |
| CON-08 | 미디어 원본은 오브젝트 스토리지, DB엔 URL+메타(JSONB)만. | 스토리지 연동 |

## 6. 데이터 모델 (최종 9테이블)

**관광 마스터 5 + 모임 2 + 여행/지도 2**:

| 테이블 | 변경/역할 | 비고 |
|--------|-----------|------|
| `users` | 현행 UserDto 스키마 확정 | CON-04 |
| `sidos`/`guguns`/`contenttypes`/`attractions` | 관광 마스터(TourAPI) | FR-TOUR |
| `trips` | user_id/group_id XOR, end_date 완료 판정, **`data` JSONB = 항목·미디어 전체 문서(GIN)** | CON-06,07 |
| `groups`/`user_group` | 모임·멤버 매핑 | CON-03 |
| **`group_region_media`** | 그룹 지도: 지역당 대표 미디어 1개(group·trip FK, 지역당 UNIQUE) | FR-MAP |

> **모델 피벗(2026-06-10, ERD.md §5)**: `trip_attractions`·`trip_region_snapshots`·`trip_media` **제거** → 항목·미디어는 `trips.data` JSONB 문서, 그룹 지도 대표만 `group_region_media` 테이블. 원칙 = "자기 여행 안에서 끝나는 데이터는 문서(JSONB), 관계·제약 있으면 테이블".

상세 DDL·인덱스·API는 [기능명세서](./기능명세서.md), 정본 스키마는 `backend/src/main/resources/db/postgres/schema.sql` 참고.
