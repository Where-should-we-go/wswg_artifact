# 디자인 시스템 — 어디갈래?(WSWG)

**버전**: v0.3 (2026-06-23 §7 반응형 정책 추가 · v0.2 2026-06-22 승인)
**무드**: 데스크탑 우선 · 반응형(모바일·태블릿) · Notion-style 블록 에디터 · 뉴트럴 페이퍼 톤
**시안(단일 소스 미리보기)**: [`design-flow.html`](./design-flow.html)
**적용 위치**: `wswg_frontend/` (Vue 3 + Vite, shadcn-vue + Tailwind 도입 예정)

> 본 문서는 프론트엔드 디자인의 **단일 소스(Source of Truth)** 입니다. 색·폰트·간격·컴포넌트 규칙을 여기서 정의하고, 코드는 이 토큰만 참조합니다. 별도 디자인 라이브러리(npm 패키지)는 만들지 않고 **프로젝트 내부 디자인 시스템**(토큰 + `components/ui/`)으로 운영합니다.

## 1. 브랜드 전략

### 1.1 포지셔닝 문장 (내부 의사결정 기준)
> **함께 떠날 곳 정하느라** 카톡에서 링크 던지고 시간 쓰는 **20·30대 그룹**에게,
> **어디갈래?**는 지역·스타일만 고르면 일정을 **자동으로 짜주고 친구와 실시간으로 같이 고치는** 협업형 여행 플래너다.
> 흩어진 검색과 의견 조율을 오가던 기존 방식과 달리, 우리는 **그걸 한 화면(노션식 문서)에서 끝낸다.**

### 1.2 타깃
- **주 타깃**: 20·30대 친구/연인 그룹 (함께 떠나고, 함께 정함)
- 함의: 협업·실시간 편집이 1급 기능. 캐주얼하지만 정보는 믿을 수 있어야 함.

### 1.3 퍼스널리티 (Aaker — Excitement + Sincerity)
| 키워드 | 정의 | Do | Don't (anti-trait) |
|---|---|---|---|
| **설레는** | 여행 전 두근거림을 건드린다 | 가볍고 들뜬 해요체, 동사로 초대 | 과장된 호들갑, **허세** |
| **친근한** | 곁에서 거들어주는 톤 | 부드러운 **해요체**, "저희/같이" | **반말** · 차갑고 **사무적**인 안내문 |
| **든든한**(보조) | 정보는 믿을 수 있게 | 명료한 데이터·근거 | 불안·**압박** 조성 |

> anti-traits 3종 — **허세 · 사무적 · 압박** — 은 어디서도 쓰지 않는다.

### 1.4 아키타입
- **주: Explorer(탐험가)** — 발견·자유 · **보조: Everyman/친구** — 누구나, 함께, 부담 없음
- 서사: *욕망=떠나고 싶다 / 두려움=계획이 귀찮다·의견조율이 힘들다 / 약속=고민은 우리가, 너흰 떠나기만 / 금기=복잡함·강요*

## 2. Voice & Tone

**Voice(불변) 4형용사**: 친근한 · 명료한 · 들뜬 · 든든한
**Tone 우선순위**: **명료성 > 재미** (헷갈리면 명확함이 먼저)
**말투 규칙**: 기본 **해요체(존댓말)**. **반말 금지.** 단 딱딱한 합쇼체·사무적 표현이 아니라 부드럽고 들뜬 해요체로 친근함을 유지.

| 상황 | Do (해요체) | Don't |
|---|---|---|
| 홈 헤드라인 | "고르기만 하세요, 일정은 저희가 짜드려요" | 반말 "…짤게" / "최적의 여행 솔루션을 경험하세요" |
| 빈 상태 | "아직 계획이 없어요! 첫 여행을 만들어볼까요?" | 반말 "…만들어볼까?" / "데이터가 존재하지 않습니다" |
| 에러 | "잠깐, 길을 잃었어요. 다시 해볼까요?" | "오류 코드 500 발생" |
| 완료/저장 | "일정이 저장됐어요! 친구를 불러 같이 다듬어보세요" | 반말 "…다듬자" / "처리가 완료되었습니다" |
| 협업 알림 | "민지님이 점심 일정을 바꿨어요" | "data.items[1] updated" |

## 3. 컬러 시스템

### 설계 원칙 (Notion-style)
1. **뉴트럴 우세** — 앱 크롬(사이드바·툴바·배경)은 무채색 페이퍼 톤이 화면의 ~90%.
2. **콘텐츠에만 색** — 색은 블록 타입 태그·상태에만. 알록달록한 카드 UI 금지.
3. **블루는 절제** — 브랜드 블루는 액션(버튼)·선택·링크에만. 면적 최소.
4. **3레이어 토큰** — primitive → semantic → component. 컴포넌트는 semantic만 참조.

### 3.6 접근성 (WCAG 2.2)
- 본문 텍스트 대비 4.5:1, 큰 텍스트 3:1 확보. `--brand`(#2C6FE3)는 흰 텍스트 버튼/링크 대비 통과.
- ⚠️ 태그 배경(soft) 위 텍스트는 반드시 짝지은 진한 ink 색 사용(위 표). 옅은 배경에 옅은 텍스트 금지.
- 색만으로 상태 전달 금지 → 아이콘/텍스트 병행 (블록 타입 = 이모지 + 라벨).
- 확정 전 axe로 전 화면 대비 스캔. 라이트/다크 두 모드 재검증(다크는 추후 토큰 오버라이드).

## 6. 컴포넌트 원칙

### 6.1 앱 셸 (데스크탑)
- 좌측 **워크스페이스 사이드바**(228px): 워크스페이스 스위처 + 검색/홈/지도 + 페이지 트리(여행=페이지, 일차=하위 페이지, 이모지 아이콘).
- 상단 **토픽바**: 브레드크럼 + 실시간 동기화 표시 + 동행인 아바타 스택 + 공유 버튼.
- 본문 **캔버스**: 커버·페이지 아이콘·타이틀·속성 테이블·뷰 탭·블록 영역.

### 6.2 핵심 = 여행 계획 블록 에디터
`trips.data` JSONB(`items[]`)를 노션식 블록으로 1:1 시각화. (도메인: [`docs/여행기록도메인.md`](여행기록도메인.md))

| data 필드 | UI 표현 |
|---|---|
| `type` (관광/식당/이동/메모/숙소) | 블록 타입 태그(§3.3) |
| `properties` (memo/budget/rating/예약번호…) | 인라인 속성 pill (`💰9,000원` `⭐5.0`) |
| `media[]` | 블록 하단 미디어 썸네일 스트립 |
| `visitDate` / `time` / `order` | 일차 그룹 + 시간 라벨 + 드래그 정렬 |
| `content_id` null | "직접 추가 · TourAPI 없음" 표기 |

- **블록 인터랙션**: hover 시 드래그 핸들 `⋮⋮` + `+`, `/` 슬래시 메뉴로 타입 선택.
- **뷰 탭**("하나의 원본 → 여러 뷰"): 📅 일정 / 🖼️ 갤러리 / 🗺️ 지도 / 📋 보드.
- **상단 속성 테이블**: 날짜·지역·동행·스타일·예산 (노션 데이터베이스 프로퍼티).

### 6.3 실시간 협업
- 동행인 **아바타 스택**(이니셜, 카렛 색과 동일).
- 다른 사용자 편집 위치 = **컬러 카렛 + 이름 태그**(`│민지`). 편집 중 블록은 해당 색 옅은 배경.
- "실시간 동기화/N명 편집 중" 상태 표시. (백엔드: Redis 수신 → batch flush)

### 6.4 그룹 지도 뷰
- 지역(시도/구군) 색칠 + **지역당 대표 추억 1개** 핀 (`group_region_media`). 우측 패널에 대표 추억 카드.

## 7. 반응형 정책 (Responsive)

### 7.1 원칙 — 단계가 디바이스를 정한다
**계획(블록 풀편집)만 데스크탑 우선, 나머지는 양쪽 동등.** 토큰·레이아웃은 데스크탑이 default, 모바일은 `max-md:` 유틸로 override.

| 단계 | 화면 | 주 디바이스 | 근거 |
|---|---|---|---|
| 탐색 | S1·S3·S4·B0 | 양쪽 동등 | 게스트 유입, 어디서나 |
| **계획** | S6 블록 에디터 | **데스크탑 우선** | 일정 풀편집·드래그·4뷰는 큰 화면이 맞음 |
| 즐기다 | 미디어 업로드 | 양쪽 동등 | 데스크탑=드래그앤드롭 / 모바일=카메라, 둘 다 1급 |
| 기억 | S8 발자취 지도 | 양쪽 동등 | 지도는 폰에서도 훌륭(풀스크린+바텀시트) |

> S6는 **데스크탑 풀편집 / 모바일은 조회+간단편집**(블록 추가·메모·사진·드래그 재정렬). 슬래시(`/`)메뉴·복잡 속성 편집은 데스크탑 우선.

### 7.2 브레이크포인트 (Tailwind 기준)
| 구간 | 폭 | 레이아웃 |
|---|---|---|
| 모바일 | `< 768` (max-md) | 1열 · 사이드바→햄버거 drawer · 필터/패널→바텀시트 |
| 태블릿 | `768–1024` (md~lg) | 2열 · 사이드바 아이콘 레일(축소) |
| 데스크탑 | `≥ 1024` (lg+) | 풀 노션 워크스페이스(228px 사이드바 고정) |

### 7.3 네비게이션 적응
- **데스크탑**: 228px 워크스페이스 사이드바 고정
- **태블릿**: 사이드바 아이콘 레일로 축소
- **모바일**: 상단 앱바 + 햄버거 **off-canvas drawer**(워크스페이스/홈/검색/지도)

### 7.4 화면별 적응 패턴
| 화면 | 데스크탑 | 모바일 적응 |
|---|---|---|
| **S6 블록 에디터** | 사이드바+캔버스+속성표+4뷰탭 | 사이드바→drawer · 속성표→접힘 시트 · 뷰탭 가로스크롤(**보드뷰 데스크탑 우선**) · 블록 `+` 버튼 명시·롱프레스 메뉴 |
| **미디어 업로드** | 드래그앤드롭 | **카메라/갤러리 직접 접근** |
| **S8 발자취 지도** | 지도+우측 대표 패널 | 풀스크린 지도 + **바텀시트** 갤러리 |
| **S3 검색** | 필터바+카드 그리드 | 필터→바텀시트, 카드 1열 |
| **S1 랜딩·인기추천** | Hero+가로 캐러셀 | 세로 스택, 캐러셀 스와이프 |
| **S5 자동생성** | 2열 폼 | 1열 폼 |
| **S7 마이페이지** | 카드 그리드 | 카드 1열, 탭 상단 고정 |
| **S9 모임** | 리스트+상세 패널 | 리스트 → 상세 push |
| **S-ADM 관리자** | 데스크탑 전용 | 모바일 미지원 허용 |

### 7.5 터치·인터랙션
- **터치 타깃 최소 44×44px**(모바일). 호버 전용 인터랙션(드래그핸들 `⋮⋮` hover 노출)은 모바일에서 **항상 보이는 `+`/`⋮` 버튼**으로 대체.
- 바텀시트·drawer는 스와이프 닫기 지원. 모달은 모바일에서 **풀스크린 시트**.

```
components/ui/               # shadcn-vue 컴포넌트 (Button, Input, Dialog, Tabs, Avatar …)
components/common/           # 직접 만든 공용 컴포넌트
features/trip/               # 블록 에디터·뷰·협업 (도메인)
features/place/              # 검색·상세
```
스텝: ① Tailwind + shadcn-vue init → ② `tokens.css`로 §3·§5 매핑 → ③ `ui/` 필요 컴포넌트만 add → ④ 블록 에디터(features/trip) 구현.

> ⚠️ `wswg_frontend`는 별도 git 레포(현재 `master`). 코드 작업 전 떠 있는 미커밋 변경(package.json/pnpm-lock) **내용 확인 후 분리 커밋** → `feature/design-system` 브랜치 분리.
